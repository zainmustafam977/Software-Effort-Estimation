# Software Effort Estimation - HCI & Scrum Analytics

This repository contains the code, datasets, and a fully responsive web application analyzing software effort estimation using historical datasets (like Albrecht, ISBSG, etc.) mapped to the Scrum Agile framework.

## Project Overview

This project was developed for a Human-Computer Interaction (HCI) course to demonstrate how traditional software engineering metrics (like Function Points and Lines of Code) can be effectively mapped to Agile Story Points for modern Sprint planning. 

It features an interactive, glassmorphic UI built with vanilla HTML, CSS, and JS, showcasing 10 advanced analysis directives including Random Forest predictions, correlations, and Sprint simulations.

## Repository Structure

- `index.html`, `styles.css`, `script.js`: The frontend web application.
- `assets/`: Contains images, generated data plots, and UI assets.
- `datasets/`: The 9 historical software effort estimation datasets (CSV).
- `scripts/`: Python scripts used to analyze the data and generate the plots (`run_analysis.py`).
- `Scrum Methodology.pdf`: The official university Agile guidelines.
- `Scrum_HCI_Proposal.docx`: The project proposal.
